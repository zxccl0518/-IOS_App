机器人相关[编辑]

因为机器人在出厂时候就会带有唯一的身份识别码，所以机器人的注册过程是厂商管理者进行的。已经注册好的机器人才有登录服务器的权限。 每个出厂的机器人都会在外包装印刷上条形码或者串号，或者二维码。 分别表示机器人的唯一ID，以及对应APP的下载地址。

机器人登录比较简单， 因为机器人的Robotid是事先在云端生成好的。所以机器人和服务器就只有登录，退出登录，以及消息发送接收。 robot ID： 000100151201000001 ～ 000100151201000020 是测试用的ID。

机器人登录[编辑]

{
  "cmd":"login",
  "robot_id":"robot id"，
  "apmac":"ap热点的mac"，
  “ssid":"ap热点的ssid",
  "ipaddr":"局域网的ipaddr"
}


机器人登录应答--异常 

{
   "status":非0,
   "content":{
       "msg":"机器人不存在"
   }
}

机器人登录应答--正常 

{
   "status":0,
   "content":{
       "is_first":1,
       "msg":"登录成功"
   }
           
}

用户管理[编辑]

用户需要下载APP，第一次使用，进入注册页面，使用用户名和密码向服务器请求注册。发送到服务器的密码是加密的，不用担心服务器端的密码泄漏。 注册成功后，可以进入关联机器人的页面，这里有2种方式可以得到想要关联的机器人为一ID。 其一， 手动扫描或者输入外包装上的条码或者数字串号；其二，可以给机器人上电后，进入配置模式，此时APP可以搜索附近的机器人，选择您的目标机器人，进行关 联。 第一次关联的APP用户将拥有机器人的管理权限。 如果不是第一次管理，用户的关联动作需要得到管理员的授权。


手机APP上的操作分为几个阶段： 注册， 登录， 退出， 交互 注册需要提供用户名和密码； 登录同样需要用户名和密码，服务器返回一个token， 时效24小时； 主动退出登录的时候，token同时失效； 交互是和服务器进行通讯的主要途径，APP需要提供的是用户名和token。

获取用户关联的机器人列表[编辑]

{
 "cmd":"message",
 "user_name":"用户名"
 "token":"令牌",
 "content":{
      "method":"query",
      "param":"robot_bind"
 }
}


返回 

   {
     "status":0,
     "content":{
       "count":关联的机器人个数,
       "robots":[关联的机器人ID数组]
       "admin":[对应于上面的机器人ID，具有管理员权限的机器人id是？ 如果没有就是0](Note:admin为字符串类型的数组)
       "online":["1","0",...] 是否在线
     }
   }

获取机器人关联的用户列表[编辑]

{
 "cmd":"message",
 "user_name":"用户名"
 "token":"令牌",
 "content":{
      "method":"query",
      "param":"user_bind", 
     "robot_id":"机器人ID" // 需要查询和此机器人相关联的用户列表
 }
}


返回 

   {
     "status":0,
     "content":{
       "count":关联的用户个数,
       "admin_count":$admin_count, // 管理员的个数
       "users":[关联的用户name数组]
       "admin":[对应于上面的用户Account，是否为机器人管理员，只能有一个管理员，如果是管理员则为1, 如果没有就是0](Note:admin为字符串类型的数组)
     }
   }

注意： 如果用户没有绑定机器人，是没有权限获取具体的用户列表的。 所以，返回的列表为空，但是用户的个数，以及管理员的个数都正常返回。

       <input type="hidden" name="toUser" value="rbt_B13CXXXXXXXXXX">

其中from参数也可能：from="rbt_CB001xxxxxx" 分别表示从机器人还是APP

错误码[编辑]

如果正确， 服务器返回 

{
 "status":0,
 "content":｛
    "msg":"OK"
  ｝
}

如果出错 status字段不为0，content字段为错误的文本内容

100开始的错误是命令相关的：

  • 100: 格式错误
  • 101: 非法命令

200开始的错误是注册相关的：

  • 200: 用户名已存在
  • 201: 用户名包含非法字符
  • 202: 密码错误

300开始的错误是其他相关：

  • 300: 用户不存在
  • 301: token非法
  • 302: token已过期
  • 303:不能重复绑定
  • 304: 机器人不存在
  • 305: 没有绑定的机器人
  • 306: 请先转让管理权限
  • 307: 您没有此权限
